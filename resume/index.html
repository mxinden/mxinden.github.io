Max Leonard Inden

Experience#

Software Engineer at Mozilla#

June 2024 - now

Core maintainer of Firefox’s QUIC / HTTP/3 networking stack: neqo (Mozilla’s Rust QUIC implementation), its integration into Firefox, and related open-source libraries. Across both Mozilla engagements: 378 pull requests and 500+ code reviews on neqo, 100+ commits to Firefox, and 17+ shepherded neqo releases (v0.9.0 to v0.25.0).

  • Multi-gigabit UDP IO: rewrite Firefox’s legacy NSPR-based UDP IO path as a modern, memory-safe Rust stack on top of quinn-udp, rolled out across Linux, Windows, macOS and Android. Multi-packet IO (recvmmsg), Generic Segmentation/Receive Offload (GSO/GRO) and a zero-allocation receive path raised CPU-bound throughput from < 1 Gbit/s to 4 Gbit/s. Built IO benchmarks with CI regression detection to drive the work.
  • Explicit Congestion Notification (ECN): implement end-to-end QUIC ECN: marking, reading, RFC 9000 path validation and Congestion Experienced feedback to the congestion controller, resilient to middlebox interference. ~50% of Firefox Nightly QUIC connections now run on ECN-capable paths.
  • Flow control: stabilize stream receive-window auto-tuning toward the bandwidth-delay product, replacing a hard 1 MB cap (~160 Mbit/s on a 50 ms link).
  • New protocols: implement MASQUE connect-udp (RFC 9298) and classic HTTP CONNECT over HTTP/3, letting Firefox proxy both TCP and UDP over a single HTTP/3 connection. Improve WebTransport support.
  • Happy Eyeballs v3: author mozilla/happy-eyeballs from scratch: a protocol-agnostic Rust state machine for dual-stack connection racing (HTTPS/SVCB records, alt-svc, ECH retry configs per RFC 9849), integrated into Firefox. Contribute to the IETF Happy Eyeballs v3 draft and present the work at IETF 125.
  • Telemetry & profiling: introduce Glean metrics and Firefox profiler markers into the HTTP3/QUIC Rust stack.
  • quinn-udp maintainer: the cross-platform UDP IO crate shared by Quinn and Firefox (45 PRs); fix platform issues (Windows ARM USO, Android EINVAL, macOS address families) that benefit the wider Rust networking ecosystem.

External contributor to Mozilla’s HTTP3/QUIC stack#

December 2023 - May 2024

89 pull requests to mozilla/neqo before joining Mozilla full-time, laying much of the groundwork for the work above.

  • UDP IO foundation: rewrite neqo’s client and server UDP IO path, leveraging sendmsg, recvmmsg and GRO via quinn-udp.
  • Binary refactor: move the client and server from mio to tokio, merge them into a single neqo-bin crate, and replace mozilla-central’s http3server custom UDP IO stack with the shared neqo code.
  • CI: add an initial QUIC Interop Runner integration.
  • Security: report and fix a security vulnerability (unbounded memory allocation from unsanitized network input, CVE-2024-2613).
  • Correctness & performance: fix cross-layer race conditions and draft stream receive-window auto-tuning, removing an upper throughput limit on high bandwidth-delay-product connections (e.g. 160 Mbit/s on a 50 ms link).
